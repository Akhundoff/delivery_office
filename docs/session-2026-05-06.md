## Fixes & Changes Made

### 1. Q.İ kodu column overflow fix

**File:** `src/modules/declarations/hooks/use-declarations-table-columns.tsx`

**Problem:** The "Q.İ kodu" (globalTrackCode) column was overflowing into the next column when the value was long.

**Fix:**

- Changed column spread from `nextTableColumns.small` → `nextTableColumns.normal` (wider fixed width)
- Replaced the raw `<Tag>` cell renderer with `OverCell` (truncates with ellipsis, expands on hover)

```tsx
// Before
{
  ...nextTableColumns.small,
  accessor: (row) => row.globalTrackCode,
  id: declarationQueryKeys.globalTrackCode,
  Header: 'Q.İ kodu',
  Cell: ({ cell: { value } }: any) => value ? <Tag>{value}</Tag> : null,
},

// After
{
  ...nextTableColumns.normal,
  accessor: (row) => row.globalTrackCode,
  id: declarationQueryKeys.globalTrackCode,
  Header: 'Q.İ kodu',
  Cell: OverCell,
},
```

## Key Architecture Reminders

- `OverCell` — use for text that may be long; truncates with ellipsis, expands on hover
- `TagCell` / `OverflowTag` — styled Tag that also handles overflow (use for track codes etc.)
- `CheckCell` / `Check` — boolean display; `Check` can be used standalone outside a table
- `SwitchCell` — when a column needs an async toggle (e.g. active/inactive)
- `PriceCell.Auto` — when currency varies per row; `PriceCell.USD` etc. for fixed currency columns
- `CountryCell` — resolves country id → name via `SettingsContext`
